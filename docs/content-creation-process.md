# Content Creation Process

How content gets created from real work, not as a separate activity.

---

## Claude Code Sessions → Content

Every Claude Code session is a potential piece of content. The workflow:

1. **Work on a real task** — trends analysis, PKM structuring, skill building, Alisa Ops, whatever the day brings
2. **Export the session** — `/export-session` generates a browsable HTML transcript
3. **Extract the use case** — pick the interesting part: a problem solved, a workflow discovered, a tool built, a hypothesis tested
4. **Write it up** — use `/stan-writing-style` for Telegram posts, or package as a longer piece for the blog

### What makes a good use case

- A real problem you hit, not a hypothetical
- The actual sequence of steps (not sanitized)
- What didn't work before what did
- A takeaway the reader can apply

### Channels for use cases

| Format | Channel | Frequency |
|--------|---------|-----------|
| Ukrainian AI/tech post | @llms_ua | When something genuinely interesting happens |
| English essay | 99rabbits.com / bloomallegedly.substack.com | Weekly or when a use case is substantial |
| Short-form video | YouTube Shorts / Reels / TikTok | See creative outlets doc |

### The key insight

Content is a byproduct of work, not a separate job. The session transcript IS the raw material. The `/export-session` skill + secret gist sharing makes it trivial to capture and share.
